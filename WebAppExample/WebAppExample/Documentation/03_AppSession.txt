[1] Application 객체
	Application 객체는 응용 프로그램 영역에서 어떤 값을 저장시켜 놓고
	그 값을 웹페이지 어느 곳에서 참조할 수 있는 기능이다.
	그래서 응용 프로그램 레벨 변수로 주로 사용되고
	Application["이름"] = 값; 의 형식으로 값을 저장한다.

	Lock() : 애플리케이션 변수를 잠그는 메서드
	UnLock() : 잠긴 애플리케이션 변수를 해제하는 메서드
	Add() : 애플리케이션 변수를 생성
	Application_Start() : 웹 애플리케이션이 시작될 때 발생 (웹 사이트에 첫 번째 사용자 방문시 발생)
	Application_End() : 웹 응용 프로그램이 끝날 때 발생(웹 사이트에서 마지막 사용자가 나간 후 발생)

[2] Session 객체
	단일 사용자 레벨로 어떤 값을 저장하거나 호출할 때 사용되는 객체로
	사용자 한명에 대한 정보와 키 값으로 저장할 수 있으며
	서버 측 메모리에 해당 데이터를 저장한다.
	Applicaton 변수가 누구나 접근이 가능한 공통적인 성격이라면 Session 변수는 사용자별로 하나씩 생성되는 전용적인 성격이다.
	Session 개체를 사용자 한명이 웹사이트를 방문했을 때 그사용자에게만 특정한 값을 저장해 놓고 유지하고 싶을 때 사용한다.
	회원 로그인 기능을 구현 할 때, 로그인했다는 정보를 보관할 때 주로 사용한다.

	SessionId : 현재 세션의 고유 번호 값 반환
	SessionTimeout : 세션 시간 기록 : 기본값은 20분
	Abandon() : 현재 세션 지우기
	Session_Start() : 사용자 한명이 방문할 때 실행 (세션 발생시 실행)
	Session_End() : 사용자 한명이 나간 후 실행 (세션 아웃 발생시 실행)

	A.aspx와 B.aspx 두 페이지 간에 데이터를 유지하고자 할 때, 
	공통 데이터는 Application 개체 변수를 사용하고
	전용 데이터는 Session 개체 변수를 통해 관리한다.

	=> Application_Start() : 웹 사이트 하나 생성시 사용자가 처음 방문했을 때 실행
	=> Session_Start() : 사용자 한명이 방문할 때 마다 실행
	=> Session_End() : 사용자 한명이 마지막 요청(접속) 후 20분 뒤에 실행
	=> Application_End() : 웹 사이트에 마지막 사용자가 나간 후 20분 뒤에 실해되는 메서드
